**Парсер для сбора, структуризации и анализа отзывов к литературе**

Программное решение для автоматизированного сбора (парсинга), глубокой технической предобработки и структурированного хранения данных с крупнейших литературных платформ Wattpad и Litnet. Система предназначена для последующего анализа популярности произведений, виральности контента и изучения читательских реакций.

**Основные компоненты**:

1. Парсеры (Scrapy, Playwright, API): Извлекают данные.
2. Предобработчики (Python): Очищают и нормализуют данные.
3. Оркестратор (Python): Управляет логикой запуска и последовательностью действий.
4. Загрузчик (Python/ODBC): Отвечает за коммуникацию с СУБД.
5. База Данных (PostgreSQL): Хранит результаты.
6. Аналитика (Jupyter): Визуализирует метрики.

**3. Модули сбора данных (Parser)**

Проект включает три независимых парсера, адаптированных под специфику площадок.

**3.1. Парсер Litnet (Разработчик: Ярослав)**

* **Технология**: Фреймворк Scrapy.
* **Особенности**: Асинхронный сбор данных за последний год.
* **Режимы работы**:

  * Simple: Разовый срез данных.
  * Jobdir: Режим с сохранением состояния (возможность паузы и докачки).
  * Offset: Пагинация с заданного смещения.
* **Предобработка**: Встроена в пайплайн обработки, выполняет базовую очистку и приведение типов.



**3.2. Парсер Wattpad №1: Scrapy-версия (Разработчик: Илья)**

* **Технология**: Scrapy + Selenium (для авторизации).
* **Логика работы**:
* **Авторизация**: Если куки в cookies.json неактуальны, запускается Selenium для ручного входа, после чего токены перехватываются и сохраняются.
* **Сбор**: Использует AJAX-запросы для получения комментариев и ответов. Режим \_replies позволяет собирать глубокие ветки обсуждений.
* **Техническая предобработка** (techpreproc.py):

  * Удаление артефактов и лишних пробелов (strip).
  * Приведение дат к стандарту PostgreSQL.
  * Конвертация числовых строк в int.



**3.3. Парсер Wattpad №2: Гибридная версия (Разработчик: Рената)**

* **Технология**: Wattpad API + Playwright + BeautifulSoup.
* **Логика работы**:
* **Гибридный метод**: Первичный запрос идет через официальное API (JSON). При достижении лимитов или отсутствии данных система переключается на Playwright (эмуляция браузера).
* **Детекция языка**: Многоступенчатая проверка через langdetect и статистический анализ (кириллица/латиница).
* **Особенности**: Автоматический поиск по 25 жанрам (тегам), интеллектуальное извлечение года публикации через регулярные выражения и мета-теги. Сохранение промежуточных результатов в JSONL и формирование текстовых отчетов со статистикой сессии.

**4. Модуль базы данных**

Данные хранятся в СУБД PostgreSQL 12+.

**4.1. База данных Wattpad (Разработчик: Данил)**

Реляционная иерархическая модель, спроектированная с учетом глубокой вложенности контента.

**Логическая модель:**

* parser\_run: Регистрация сеанса сбора данных (время начала, runtime). Позволяет идентифицировать конкретный цикл парсинга.
* user: Профили пользователей платформы (авторы и читатели).
* book: Метаданные произведения (название, описание, голоса, просмотры, расчетное время чтения).
* chapter: Данные о главах (название, ссылка, дата публикации, статистика вовлеченности).
* tag: Справочник жанров и тегов для классификации произведений.
* book\_by\_tag: Реализация связи «многие-ко-многим» между книгами и тегами.
* comment: Тексты отзывов. Поле parent\_id реализует древовидную структуру (поддержка ответов на комментарии).
* rank: Позиция книги в различных рейтингах (чартах) платформы на момент сбора.

**4.2. База данных Litnet (Разработчик: Андрей)**

Реляционная модель, оптимизированная под хранение снапшотов.

* parser\_run: Якорная таблица снапшота. Хранит время запуска (run\_at).
* book: Срез данных книги (рейтинг, лайки, просмотры, цикл, цена, статус завершенности, times\_saved\_to\_library).
* chapter: Список глав с датами публикации.
* comment: Тексты отзывов с привязкой к книге и пользователю.
* reward: Информация о полученных книгой наградах (type, amount).
* Справочники: user (авторы/читатели), genre, tag.
* Таблицы связей (со снапшотами): books\_genres (хранит top\_position), books\_tags, books\_users.

**4.3. Целостность и стандарты**

* **Идентификация**: Использование UUID для всех первичных и внешних ключей.
* **Типизация**: Использование TIMESTAMP WITH TIME ZONE для корректной работы с датами и INTERVAL для временных промежутков.
* **CASCADE**: При удалении книги автоматически удаляются все её динамические зависимости (главы, комментарии, награды, позиции в рейтингах).
* **RESTRICT**: Запрещено удаление справочных записей (пользователей), если на них ссылаются существующие снапшоты данных.

**5. Управляющие модули и Загрузка**

**5.1. Оркестратор (Разработчик: Михаил)**

* **Модуль**: dev/orchestrator/.
* **Функция**: Точка входа в ETL-процесс. Читает конфиги, инициирует сбор данных и после завершения автоматически вызывает Загрузчик.
* **Цель**: Исключение ручных операций между сбором и записью в БД.

**5.2. Загрузчик (Разработчик: Андрей)**

* **Модуль**: upload.py, models.py.
* **Функция**: Реализует соединение с БД и сохранение объектов класса Books. Управляет сессиями через parser\_run, синхронизирует справочники (теги, авторы) и записывает актуальные срезы данных о книгах.
* **Цель**: Обеспечение целостности данных и создание временных снапшотов для анализа динамики популярности.

**7. Аналитика (Разработчики: Екатерина, Марина, Ярослав)**

Цель модуля – обоснование популярности книг на основе культурных и возрастных особенностей ЦА.

* **Модуль**: /Analytics/ (Jupyter Notebooks).
* **Функция**: Обоснование популярности книг через лингвистический и статистический анализ отзывов с применением теории культурных измерений Хофстеде. Модуль сопоставляет метрики БД с культурными кодами двух групп ЦА: женщин 40–60 лет (СНГ, Litnet) и девушек 18–24 лет (США/ЮА, Wattpad).

**Методология**:

Для Litnet анализируется запрос на «эмоциональную безопасность» (высокое избегание неопределенности), долгосрочный путь к счастью и ценность заслуженного успеха.

Для Wattpad исследуется фокус на краткосрочном драйве (клиффхэнгеры), соревновательных тропах (Enemies to Lovers) и праве на удовольствие без цензуры.

**Цель**:

* Выявление ТОП популярных книг и жанров через корреляцию метрик БД.
* Определение трендов в отзывах (анализ ключевых триггеров внимания).

**Масштабирование**: интеграция БД с BI-инструментами (Power BI / Tableau).
